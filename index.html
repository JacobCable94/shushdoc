Introduction

I tried to follow some sensible conventions, however as my first project in React Native and javascript/typescript in general, I'm aware there are probably things I have missed, or better practices. Any questions just message me on Slack or we can video call!

Summary of component tree

We have the "entry file" ./index.js. This imports ./App.tsx, where the app is wrapped in navigation and theming providers. App.tsx imports /src/GlobalNav.tsx, which renders a video backgorund and then allows navigation between four screens: WelcomePage, EnterPinPage, SignupNav, LoggedInNav.

Navigation

We have three navigation providers using React Navigation. GlobalNav screens conditional on existingUser and authenticated context state values, SignupNav shows onboarding flow, LoggedInNav renders the logged in screens of the app (home, card etc).

I used custom navigators, as it seemed the easiest way to obtain a transparent background for the screens (to leave the video background component visible). These custom navigators are very minimal. I didn't find the documentation on react navigation for custom navigators very helpful.

Layouts

There are three layouts currently, one for logged in, one for signup, an extra one for the camera page (ID&V)

SignupLayout

This has an "offset mode" used by PrepPage, and a none-offset mode.

LoggedInLayout

this has various headers depending on the page

CameraLayout

A specific layout for the camera page

Components

Shadow

This component was made to wrap other components and deal with shadows. I had some issue trying to implement a "glow" with colour.

Background

This component uses the module react-native-video to render the motion-background asset provided by Patrick. It is rendered once in the GlobalNav component.

BalanceDisplay

This component is a View which displays account balance. It has two subcomponents AccountSummary and MoneyInandOut

Buttons

There are lots of button components. The index.tsx file here exports them all as one master component "ShushButton"

DropDownMenu

This component is used in LoggedInLayout when the "page" property has value "Home". On the home page we want a parent to be able to select child accounts via a dropdown menu, to check their balance etc.

Navigators

This subdirectory contains two basic custom navigator components using react navigation v5.x. See Navigation for more info.

Numpad

This is a pin pad component.

SelectionMenu

This is just a menu of buttons, similar to UserSelection, should probably be merged with that if it is used.

Task

This is a component to display a task, with (as yet unfunctional) buttons to request, complete etc.

UserSelection

This component is just a menu to select child users/dependent accounts for some action. Should probably be merged with SelectionMenu.

Container

This was an early component I made just to place other components and content in position "absolute" at a certain percentage down the Screen

Divider

A very simple component to render a thin grey divider between bits of content.

ProgressBar

Work-in-progress attempt to track progress through onboarding/sign up flow.

ShushCard

A component to display a virtual debit card

ShushIcon

Icon component. I was having trouble converting Patrick's svg icons to a font for RN. My solution was just to use react-native-svg and input a stripped-down version of the xml file for each icon.

ShushWritten

This component renders the brand name logo "Shush" in white.

Subtitle

A subtitle component (I was reusing the same code in places so refactored it here)

Page components in Signup Navigation

PrepPage

Page which displays what a user needs to sign up (ID if they're an adult)

DobPage

collects date of birth

PhonePage

collects phone number, navigating away is conditioned on user age (as only adults need IDV check)

IDVSelectPage

only rendered for adult users, choosing type of IDV (all of which just navigate to the camera page)

IDVCameraPage

basic camera page using RNCamera

NamePage

collects name

EmailPage

collects email (rendered for adults only)

AddressPage

collects address

ConfirmationPage

displays all collected data, should allow users to change info before sent off

TnCPage

displays TnC

SetPinPage

Setting a Pin (via react-native-keychain)

Page components in "Logged in" Navigation

HomePage

Home Page, has subpages for different cards/accounts, has dropdown menu to view children accounts

CardPage

Card settings page for shush-now card

EarnPage

Task/Chore system (needs to be slightly different for parent/child)

GrowPage

Shush stars (credit score system), eventually contains spending categories

SettingsPage

Settings page

Known Issues

Typescript

I converted the project to typescript part way through, and there are currently lots of "any" types, so some of it needs "typing".

Refactoring

I did a lot of refactoring as I was learning, but things like the various "Button" components are probably suboptimal.
